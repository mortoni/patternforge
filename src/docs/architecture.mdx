## Folder map

```
src/
├── app/           # Routes and layouts
├── features/      # Domain UI (training, sets, mistakes, progress)
├── components/    # Shared UI and marketing
├── db/            # Dexie setup, schemas, seed
├── repositories/  # IndexedDB access
├── services/      # Cross-cutting orchestration
├── lib/           # Pure helpers
└── domain/        # Conceptual types and import schemas
```

Prefer **`src/db/schema.ts`** types when touching persisted data. See [State & Persistence](/docs/architecture/persistence).

## State management

No global Redux store. PatternForge uses:

- **Component state** — `useState` / `useReducer` for local UI
- **Context** — settings, theme, shell layout
- **Repositories + hooks** — read/write Dexie on the client

State transitions in training should stay traceable — one owner per phase beats scattered effects.
