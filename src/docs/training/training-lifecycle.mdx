# Training Lifecycle

**Why this exists:** PatternForge training is a sequence of connected states — not a flat list of puzzles. This page describes the full path from choosing a set to completing cycles and reviewing progress.

## At a glance

```
Training Set
    └── Cycle 1 (active → completed)
            └── Session(s)
                    └── Exercise attempts
    └── Cycle 2 …
    └── Reflection (Progress) reviews completed cycles
```

Woodpecker-style training repeats the **same set** across cycles. Each cycle is a measurable pass; recognition improves as cycle time drops.

## Phase 1 — Choose a set

A **training set** is a fixed collection of exercises loaded from static JSON into IndexedDB.

- Ordered by `exerciseIds` on the set record
- Difficulty: `easy`, `intermediate`, `advanced`, or `custom`
- Set size = length of `exerciseIds`

**Route:** `/app/sets` — browse sets, view history, start a new cycle.

Only one **active** cycle per set in normal use. Starting a new cycle after completion increments `cycleNumber`.

**Route:** `/app/training` — execution mode. Loads the current exercise when an active cycle exists. No active cycle → empty state with links to **Training Sets** or **Progress**.

## Phase 4 — Solve exercises (attempts)

Each **exercise** is one puzzle (FEN, side to move, solution line).

Each **attempt** records a single try:

- **Result** — `correct`, `incorrect`, or `skipped`
- **Duration** — `durationMs`
- **Moves** — `userMoves` in UCI (first-move mode stores one move)

Training compares your move to the expected **first move** — not the full variation. See [Design Decisions](/docs/decisions).

Failed positions can enter the **mistake review** queue for focused re-practice.

## Phase 5 — Complete the cycle

When the last exercise is solved (or the cycle flow finishes):

1. Cycle run status → **`completed`**
2. `totalTimeMs` and progress fields are finalized
3. Optional redirect to **cycle summary** for that run

The app does **not** auto-open an old summary when you return to Training without an active cycle — execution and review stay separate.

## Phase 6 — Reflect and repeat

**Progress** (`/app/progress`) shows either:

- Your **current active cycle**, or
- **Reflection** — completed cycles per set (table or chart)

Reflection is scoped **per training set** so trends are not mixed across unrelated sets. See [Reflection](/docs/training/reflection).

When ready, start **cycle N+1** on the same set and aim to reduce total time while keeping quality high.

## End-to-end flow

```
1. Open Training Sets → pick a set
2. Start cycle → cycleRun (active)
3. Enter Training → session (active)
4. Solve puzzles → exerciseAttempts recorded
5. Leave or finish → session closed
6. Complete last exercise → cycleRun (completed)
7. Review in Progress → Reflection
8. Start next cycle → cycleNumber + 1
```
