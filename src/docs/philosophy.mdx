# Philosophy

**Why this exists:** Most tactical training is consumed like entertainment — solve, move on, forget. PatternForge is built around returning to the same ideas until recognition becomes automatic.

## The problem with novelty

Puzzle apps reward rapid consumption. Immediate feedback feels productive, but it rarely builds lasting pattern memory.

## The PatternForge approach

Training is organized around **cycles** — repeated passes through a fixed set of exercises. Each cycle aims to:

- Reduce total solving time
- Increase recognition speed
- Build automatic pattern memory

The goal is not to collect solved positions. The goal is to change what you see over the board.

## From calculation to recognition

First encounter: you calculate. With repetition, calculation becomes recognition — key squares, motifs, and continuations surface faster. That transfer matters under time pressure.

## Built for real play

PatternForge targets practical board vision, not puzzle streaks. You are learning to **see patterns faster**, not to solve puzzles faster.
