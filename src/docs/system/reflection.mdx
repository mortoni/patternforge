# Reflection

**Why this exists:** After cycles complete, PatternForge separates **doing** (`/app/training`) from **looking back** (`/app/progress`) so review never hijacks execution.

## When reflection appears

**Progress** shows:

- **Current cycle** — when an active cycle exists for your context
- **Reflection** — when no active cycle (browsing completed work)

## Per-set history

Reflection is scoped **per training set**, not mixed across unrelated sets.

1. Choose a set from the selector
2. View completed cycles as a **table** or **line chart**
3. Chart plots **cycle number vs total time** (needs ≥ 2 completed cycles)
4. A **median** line is computed from that set’s data only

## Drill-down

Table rows link to **cycle summary** for a specific run — timing, counts, and session detail. Summary is opt-in from Progress; it is not forced when opening Training.
